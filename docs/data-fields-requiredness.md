# Project Data Fields: ICD, OPS, Hospital

This file lists fields currently used in the project for ICD, OPS, and Hospital data.

Legend:
- **Needed for project = Yes**: required by current ingestion flow, constraints, or keys.
- **Needed for project = No**: not required to successfully ingest/store core records.
- **Ingested = ❌**: this data is not ingested by the parser.

## ICD Data

### Source TXT Columns (input format)

Format in parser: `level|id|?|?|?|code|?|label`

| Source Column | Ingested | Needed for project | Why |
|---|---|---|---|
| `level` | Yes | Yes | Used as filter (`level == "5"`). |
| `id` | ❌ | No | Parsed positionally but not stored or used downstream. |
| `?` (3rd col) | ❌ | No | Ignored by parser. |
| `?` (4th col) | ❌ | No | Ignored by parser. |
| `?` (5th col) | ❌ | No | Ignored by parser. |
| `code` | Yes | Yes | Core identifier for ICD record and uniqueness with `version_year`. |
| `?` (7th col) | ❌ | No | Ignored by parser. |
| `label` | Yes | Yes | Required semantic label; empty labels are skipped. |

### Normalized ICD Record / DB Fields

| Field | Needed for project | Why |
|---|---|---|
| `code` | Yes | `NOT NULL`; part of unique key `(code, version_year)`. |
| `label` | Yes | `NOT NULL`; user-facing ICD text. |
| `category3` | Yes | `NOT NULL`; used for grouping/indexing. |
| `parent_code` | No | Nullable hierarchical helper. |
| `parent_id` | No | Nullable self-reference for hierarchy linking. |
| `level` | Yes | `NOT NULL`; hierarchy depth used in load order. |
| `version_year` | Yes | `NOT NULL`; part of unique key `(code, version_year)`. |
| `is_terminal` | No | Has default; derived convenience flag. |
| `code_type` | No | Has default (`primary`); classification helper. |
| `id` | Yes | Primary key generated by DB. |
| `created_at` | No | DB metadata with default timestamp. |
| `updated_at` | No | DB metadata with default timestamp. |
